SWOT Analysis — DMart Sales Analytics Dashboard
Strengths (Internal)
  • retail_rolap star schema already built — no new data infra needed
  • 2+ years of historical transaction data available
  • Defined ETL pipeline with daily refresh (agg_* tables ready)
  • RFM segments pre-computed in dim_customer
  • 350+ stores = statistically rich dataset for benchmarking
Weaknesses (Internal)
  • No real-time data — daily refresh only, 6AM cutoff
  • Online channel data missing from retail_oltp entirely
  • Power BI Gateway dependency on IT team for setup
  • No predictive or ML capability in current scope
  • Store Managers limited to their store_key — cross-store view blocked by RLS
Opportunities (External)
  • 35+ analyst hours/month freed up — can redirect to strategic work
  • Tier 2/3 city expansion planning enabled via city tier revenue charts
  • Loyalty ROI quantification can inform marketing budget decisions
  • Cross-sell opportunities from RFM "Needs Attention" segment
  • Mobile Power BI rollout for field Regional Managers on the move
Threats (External)
  • MySQL Gateway failure = zero data refresh — no fallback automated
  • Scope creep risk rated HIGH (per risk register) post sign-off
  • Low stakeholder adoption if training sessions not conducted
  • Data quality issues in retail_rolap can erode CFO trust in numbers
  • Power BI Pro/Premium licensing cost if org-wide rollout needed
